<H2>Welcome To Libero</H2>

<P><IMG SRC="leebehro.gif" ALT="LIBERO [LEE-beh-ro]" ALIGN="BOTTOM">
<B>1.</B><I>v.(latin)</I> &quot;I free&quot;, from <I>liberare</I>;
<B>2.</B><I>n.(french)</I> Go-fer for football (soccer) team, keeps things
moving during matches;<B>3.</B><I>n.</I> Brand of diapers in Europe.
&quot;Liber-o&quot; is pronounced like English &quot;liber-ty&quot; or
French &quot;liber-t&eacute;&quot;.

<P>Are you a programmer?  Do you sincerely want to write better programs?
Then take a look at Libero, a free software tool from iMatix.

<P><EM>How do I use Libero?</EM>
<OL>
<LI>Design your program visually as a state diagram;
<LI>Choose your programming language;
<LI>Generate a framework for your program;
<LI>Fill-in the framework to get from rapid prototype to working program;
<LI>Repeat until your program is perfect.
</OL>

<P><EM>What Languages can I use?</EM>
<UL>
<LI>$(*ansic=ANSI C)
<LI>$(*php=PHP)
<LI>$(*java=Java)
<LI>$(*cpp=C++)
<LI>$(*perl=Perl)
<LI>$(*awk=Awk)
<LI>UNIX shells - $(*ksh=Korn shell), $(*bash=BASH),
$(*bsh=Bourne shell), $(*csh=C shell)
<LI>$(*rexx=Rexx)
<LI>$(*vb=MS Visual Basic)
<LI>$(*setup_basic=MS Test Basic)
<LI>$(*cobol=COBOL)
<LI>$(*plsql=PL/SQL)
<LI>$(*80x86=MS 80x86 Assembler)
</UL>
  ... with open-ended support for other languages.  Libero uses a
programmable code generator engine.  To generate code in a specific language,
you write a new <I>schema</I>.  A schema is like a script or program, written
using Libero's schema language.

<P><EM>How Do I Get Libero Onto My Computer?</EM>
<UL>
<LI>Libero runs on Windows, MS-DOS, VAX/VMS, UNIX.
<LI>It is supplied with full portable ANSI C sources.
<LI><A HREF="lrinst.htm">Fast link to <EM>Installing Libero</EM>.</A>
</UL>

<P><EM>For What Can I use Libero?</EM>
<UL>
<LI>All complex programs that risk looking like spaghetti,
<LI>Realtime programming,
<LI>Mulithreaded programming,
<LI>Software robots,
<LI>Communications clients and servers,
<LI>Games,
<LI>Install scripts,
<LI>Parsers.
</UL>

<P><EM>Why do I want to use Libero?</EM>
<OL>
<LI>You can write complex programs faster and better.
<LI>Your work is easier to understand, maintain, and reuse.
<LI>You are liberated from language-dependent styles.
</OL>

<P>Libero comes from a decade of research into ways of making programs easier
to write and maintain.  It's really quite simple:<I> if you want to write
better programs, you should take a look at Libero</I>.

<P><EM>Can I Buy Libero?</EM>

<P>You can buy a commercial license for Libero from iMatix.  This license
gives you the contractual right to support and updates.  At the same time,
Libero remains free software distributed according to the terms of the
$(*gnugpl=GNU general public license).  For details about commercial
licenses for Libero, $(*info=contact us).

<P>Libero is sometimes called 'freeware' or 'shareware'.  This is not
accurate: Libero is <EM>free software</EM>.  We (or anyone) can choose
to charge a fee for Libero.  This is not our concern.  However, the
source code that makes-up Libero can never, in any form, be considered
as propietary.  It is, and will always remain, free.  If someone makes
a better version, based on those sources, that improvement falls under
the same terms as Libero.  Thus, all improvements to Libero are for the
benefit of all.

<P><EM>How Does The GNU GPL Affect My Work?</EM>

<P>The applications you develop with Libero are your concern.  In no
way do we attempt to oblige you to apply the GNU GPL to your work.  Of
course, you are also free to do so.  The code generated by Libero does
not bear any copyright statement, and is not covered by the GNU GPL,
any more than the code generated by a compiler belongs to the company
that wrote the compiler.

<P>If you include parts of the Libero source code in a development,
you are making a 'derived product', which falls under the GNU GPL.  If
you need to combine such a derived product with non-derived work, we
recommend that you separate the development cleanly.

.page Recent Changes to Libero

<H2>$(TITLE)</H2>

<P>Version 2.30, released on 8 Jun 1997;
<UL>
<LI>Added :do event local, and :do module local constructs.
<LI>Fixed a problem in the Java schema.
<LI>Help screens are no longer paginated.
<LI>Improved upper/lower-case filename generation under MS-DOS.
</UL>

<P>Version 2.22, released on 25 Feb 1997:
<UL>
<LI>If source program contained same module twice, caused code
generator to miscount the number of new module stubs.
</UL>

<P>Version 2.21, released on 1 January 1997:
<UL>
<LI>Under OS/2, uses -dpath for all files.
</UL>

<P>Version 2.20, released on 1 January 1997:
<UL>
<LI>C++, Java, Rexx schemas now a standard part of Libero.
<LI>We use event_t instead of t_event in C and C++ schemas.
<LI>Events now defined as 'enums' in C and C++ schemas.
<LI>New option "-opt:class='SomeClass'" for C++ schema.
<LI>New switch, "-dpath" for dialogs and .ini files.
<LI>New switch "-pretty" 'pretty printer', rewrites dialog.
<LI>New switch "-stats" prints dialog statistics.
<LI>New switch "-nosort" for module stubs in natural order.
<LI>In dialog, '->' is optional if next state is empty.
<LI>In dialog, event name '$other' means all other events.
<LI>UNIX and VMS binary kits no longer supported.
<LI>-help=no and -help=false now accepted as -nohelp.
<LI>:echo output sent to stderr instead of stdout.
<LI>Libero checks you don't specify the same state twice.
<LI>Libero checks against initial state as a 'next state'.
<LI>Fixed some segment violation errors in LRSORT and LRCODE.
<LI>Fixed some minor bugs in the code generator.
<LI>Fixed some minor bugs in the dialog parser.
<LI>Fixed some minor bugs in the command-line option code.
</UL>
